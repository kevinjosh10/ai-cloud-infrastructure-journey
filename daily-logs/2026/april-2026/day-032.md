### 2️⃣ Learned DNS Resolution Process

Step-by-step flow:

1. Browser cache check  
2. OS cache check  
3. Recursive resolver (ISP / public DNS)  
4. Root nameserver  
5. TLD nameserver (.com, .org, etc.)  
6. Authoritative nameserver  
7. IP returned to browser  

Key Insight:
- Each layer helps locate the exact IP of a domain efficiently  

---

### 3️⃣ Learned DNS Record Types

#### A Record
- Maps domain → IPv4 address  

#### AAAA Record
- Maps domain → IPv6 address  

#### CNAME Record
- Alias of one domain to another  

Example:
    www.example.com → example.com  

#### MX Record
- Handles email routing  

#### TXT Record
- Used for verification and security (SPF, DKIM)  

#### NS Record
- Specifies authoritative nameservers  

#### SOA Record
- Contains domain authority and configuration details  

---

### 4️⃣ Practiced DNS Commands

#### Basic Lookup
    nslookup google.com  

#### Detailed Query
    dig google.com  

#### Mail Records
    dig google.com MX  

#### Full DNS Trace
    dig +trace google.com  

Observed:
- DNS resolution path from root → TLD → authoritative server  
- Understood how DNS queries travel across the internet  

---

### 5️⃣ Understood TTL (Time To Live)

- TTL defines how long DNS records are cached  

Example:
- TTL = 300 → cached for 5 minutes  

Insights:
- Low TTL → faster updates, more queries  
- High TTL → better performance, slower updates  

Use Case:
- Reduce TTL before deployment changes  
- Increase TTL after stability  

---

### 6️⃣ Learned AWS Route 53 Basics

- AWS managed DNS service  

#### Hosted Zone
- Container for domain records  

#### Record Types
- Supports A, AAAA, CNAME, MX, etc.  

#### Routing Policies

- Simple → single resource  
- Weighted → traffic split  
- Latency-based → nearest server  
- Failover → backup system  
- Geolocation → based on user location
