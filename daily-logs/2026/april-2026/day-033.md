## ✅ What I Completed

### 1️⃣ Learned DHCP (Dynamic Host Configuration Protocol)
- Understood how devices automatically receive IP addresses
- Eliminates the need for manual IP configuration
- Used in all modern networks (home, office, cloud)

---

### 2️⃣ Understood DHCP Process — DORA
- **Discover:** Client broadcasts to find DHCP server
- **Offer:** Server offers an available IP address
- **Request:** Client requests the offered IP
- **Acknowledge:** Server confirms and assigns the IP

📌 This 4-step process happens instantly when connecting to a network

---

### 3️⃣ Learned What DHCP Provides
- IP Address (e.g., 192.168.1.10)
- Subnet Mask (defines network range)
- Default Gateway (router IP)
- DNS Server Addresses (for domain resolution)
- Lease Time (valid duration of assigned IP)

---

### 4️⃣ Understood NAT (Network Address Translation)
- Translates **private IP → public IP**
- Allows devices in private networks to access the internet
- Essential because private IPs are not routable on the internet

---

### 5️⃣ Learned Types of NAT

- **Static NAT (1:1):**
  - One private IP mapped to one public IP
  - Used for hosting servers

- **Dynamic NAT (Many:Many):**
  - Uses a pool of public IPs
  - Devices get assigned dynamically

- **PAT / NAT Overload (Many:1):**
  - Multiple devices share one public IP
  - Uses port numbers to differentiate traffic
  - Used in home routers and most real-world networks

---

### 6️⃣ Connected to AWS Cloud

- AWS VPC uses DHCP to assign IPs to EC2 instances
- Instances in private subnets do not have direct internet access
- NAT Gateway enables outbound internet access for private resources

📌 Flow:
Private EC2 → NAT Gateway → Internet

---

## 🧠 Key Concepts I Understood

- DHCP automates network configuration
- DORA is the core DHCP workflow
- NAT enables communication between private networks and the internet
- PAT is the most commonly used NAT type
- AWS uses DHCP and NAT Gateway internally for networking

---

## ⚡ Real-World Understanding

When I connect to WiFi:
- DHCP assigns my IP and network settings automatically
- NAT translates my private IP to a public IP for internet access

In AWS:
- EC2 instances get IPs via DHCP
- NAT Gateway allows private instances to access the internet securely
