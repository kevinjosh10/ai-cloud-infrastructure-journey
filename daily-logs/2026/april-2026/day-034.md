# Day 034 – Network Troubleshooting Commands

**Date:** 04 April 2026  
**Duration:** ~2–2.5 Hours  
**Focus:** Debugging Connectivity & Network Issues in Cloud Environments  

---

## 🎯 Objective

Understand how to troubleshoot network issues in real-world cloud environments.  
Learn how to test connectivity, trace network paths, inspect open ports, and debug application responses from an EC2 instance.

---

## ✅ What I Completed

### 1️⃣ Mastered `ping`
- Used `ping -c 4 8.8.8.8` to test internet connectivity
- Used `ping -c 4 google.com` to test DNS resolution
- Measured latency (RTT) and packet loss
- Verified whether remote hosts are reachable

---

### 2️⃣ Learned `traceroute`
- Installed using `sudo apt install traceroute -y`
- Used `traceroute google.com` to track packet path
- Observed multiple hops between EC2 and destination

Example:
    1  172.31.x.x  
    2  AWS network  
    3  ISP backbone  

---

### 3️⃣ Explored `ss` Command
- Used `ss -tulpn` to list active ports and services
- Filtered using `ss -tulpn | grep :80`

Observed:
    :22 → SSH  
    :80 → Nginx  

Confirmed:
- SSH service running on port 22  
- Web server (nginx) running on port 80  

---

### 4️⃣ Learned `curl`
- Used `curl localhost` to test local server response
- Used `curl http://<public-ip>` to test external access
- Observed raw HTML response from server

Example:
    HTTP/1.1 200 OK  

---

### 5️⃣ Performed Hands-On Practice on EC2

Commands used:
    ping -c 4 8.8.8.8  
    ping -c 4 google.com  
    traceroute google.com  
    ss -tulpn  
    ss -tulpn | grep :80  
    curl localhost  
    curl http://13.203.228.242  

Confirmed:
- EC2 has internet connectivity  
- DNS resolution is working  
- Network path is traceable  
- Nginx is running and accessible  
- Website is live and responding  

---

## 🧠 Key Concepts Learned

- `ping` for connectivity and latency testing  
- `traceroute` for identifying network path and failures  
- `ss` for inspecting open ports and services  
- `curl` for testing server and API responses  
- Layered troubleshooting approach in cloud systems
