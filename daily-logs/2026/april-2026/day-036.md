## ✅ What I Completed

### 1️⃣ Learned HTTP Request Structure

An HTTP request is sent from a client to a server.

**Components:**

- **Method:** Defines the action  
  - GET → Retrieve data  
  - POST → Send data  
  - PUT → Replace resource  
  - PATCH → Partially update resource  
  - DELETE → Remove resource  

- **URL:** Endpoint of the resource  
  Example: https://api.example.com/users

- **Headers:** Metadata about the request  
  Examples:  
  - Content-Type: application/json  
  - Authorization: Bearer token  

- **Body:** Contains data (used in POST/PUT/PATCH)  

Example Body:  
{ "name": "Kevin", "role": "Engineer" }

---

### 2️⃣ Learned HTTP Response Structure

A server responds back to the client after processing a request.

**Components:**

- **Status Code:** Indicates result of the request  
- **Headers:** Additional metadata  
- **Body:** Actual response data  

---

### 3️⃣ Memorized Important Status Codes

**Success:**
- 200 OK  
- 201 Created  

**Redirection:**
- 301 Moved Permanently  
- 302 Found  

**Client Errors:**
- 400 Bad Request  
- 401 Unauthorized  
- 403 Forbidden  
- 404 Not Found  

**Server Errors:**
- 500 Internal Server Error  
- 502 Bad Gateway  
- 503 Service Unavailable  

---

### 4️⃣ Understood Stateless Nature of HTTP

- HTTP does not store client state between requests  
- Each request is independent  

---

### 5️⃣ Learned Cookies & Sessions

**Cookies:**
- Stored in browser  
- Sent with every request  

**Sessions:**
- Stored on server  
- Identified using session ID (via cookies)  

---

### 6️⃣ Learned HTTPS Basics

- HTTPS = HTTP + SSL/TLS  
- Encrypts communication between client and server  
- Protects sensitive data like passwords  

---

### 7️⃣ Practiced Using curl

GET Request:  
curl https://httpbin.org/get  

POST Request:  
curl -X POST -H "Content-Type: application/json" -d '{"key":"value"}' https://httpbin.org/post  

Check Headers:  
curl https://httpbin.org/headers

## 🧠 Key Takeaways

- HTTP is the foundation of web communication  
- Requests and responses have structured formats  
- Status codes are critical for debugging  
- HTTP is stateless → requires cookies/sessions  
- HTTPS ensures secure communication  
- curl is a powerful tool to test APIs  

---

## 🚀 Real-World Relevance

- APIs and microservices use HTTP/HTTPS  
- Load balancers manage HTTP traffic  
- Debugging systems requires understanding status codes  
- Security in cloud depends on HTTPS (TLS encryption)
