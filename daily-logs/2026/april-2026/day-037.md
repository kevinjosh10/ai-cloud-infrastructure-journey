### 2️⃣ Understood HTTPS

- HTTPS = HTTP + TLS  
- Ensures:
  - Data encryption  
  - Data integrity  
  - Authentication  

- Prevents:
  - Man-in-the-middle attacks  
  - Data theft  
  - Unauthorized modification  

---

### 3️⃣ Learned TLS Handshake Process

#### Step 1: ClientHello
- Client sends:
  - Supported TLS versions  
  - Cipher suites  
  - Random number  

#### Step 2: ServerHello
- Server selects:
  - TLS version  
  - Cipher suite  
- Sends server random number  

#### Step 3: Certificate
- Server sends certificate containing:
  - Public key  
  - Domain name  
  - Issuer  

#### Step 4: Key Exchange
- Client verifies certificate  
- Generates session key  
- Sends encrypted key to server  

#### Step 5: Finished
- Both share same session key  
- Encrypted communication begins  

---

### 4️⃣ Learned Certificates

- Certificate acts as a digital identity of a website  
- Ensures server authenticity  

Contains:
- Domain name  
- Issuer (Certificate Authority)  
- Expiry date  
- Public key  
- Digital signature  

---

### 5️⃣ Understood Certificate Authorities (CA)

- Trusted organizations that issue certificates  

Examples:
- Let's Encrypt  
- DigiCert  
- GlobalSign  

- Browsers trust certificates signed by these authorities  

---

### 6️⃣ Learned Self-Signed Certificates

- Created by server itself  
- Not trusted by browsers  
- Used in:
  - Development  
  - Internal systems  

---

### 7️⃣ Compared TLS 1.2 vs TLS 1.3

- TLS 1.3 is faster and more secure  
- Requires fewer handshake steps  
- Removes weak cryptographic algorithms  
- Used in modern cloud infrastructure
