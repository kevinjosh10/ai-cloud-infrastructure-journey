---

### 2️⃣ Understood Stateful vs Stateless Firewalls

#### Stateful Firewall
- Remembers active connections  
- Automatically allows return traffic  
- More intelligent and widely used  

Example:
- Allow inbound HTTP → response automatically allowed  

---

#### Stateless Firewall
- Does not remember connections  
- Each request must match a rule  
- Requires explicit rules for both directions  

---

### 3️⃣ Learned Ingress vs Egress Rules

#### Ingress (Inbound)
- Traffic coming into the server  

Examples:
- SSH (port 22)  
- HTTP (port 80)  

---

#### Egress (Outbound)
- Traffic leaving the server  

Examples:
- API requests  
- Software updates  

---

### 4️⃣ Learned iptables Basics (Linux Firewall Core)

Command used:
    sudo iptables -L  

Understood:
- Lists all firewall rules  

---

#### Chains in iptables

INPUT  
- Incoming traffic  

OUTPUT  
- Outgoing traffic  

FORWARD  
- Traffic passing through system  

---

### 5️⃣ Learned UFW (Uncomplicated Firewall)

- Simplified firewall tool built on top of iptables  
- Easier to configure and manage  

---

#### Commands Practiced

Enable firewall:
    sudo ufw enable  

Allow SSH:
    sudo ufw allow 22  

Allow HTTP:
    sudo ufw allow 80  

Check status:
    sudo ufw status  

---

### 6️⃣ Understood AWS Security Groups

- Acts as a virtual firewall for EC2 instances  
- Works at instance level  
- Controls inbound and outbound traffic  

---

#### Key Concept

- Security Groups are STATEFUL  
- If inbound traffic is allowed → response is automatically allowed  

---

#### Common Rules

SSH → Port 22  
HTTP → Port 80  
HTTPS → Port 443
