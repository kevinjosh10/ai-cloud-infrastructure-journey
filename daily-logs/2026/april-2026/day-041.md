### 1️⃣ Learned Troubleshooting Methodology

Understood the correct debugging order:

1. DNS Issue?  
2. Routing / Network Issue?  
3. Firewall / Security Issue?  
4. Application Issue?  

💡 Key Insight:
- Always troubleshoot from **outside → inside**  
- If DNS fails, nothing else works  

---

### 2️⃣ Practiced DNS Debugging using `dig`

Command:
    dig +short api.example.com

✔ Learned:
- Converts domain → IP address  
- No output → DNS issue  
- Wrong IP → misconfigured DNS  

---

### 3️⃣ Tested Connectivity using `telnet`

Command:
    telnet host port  

Example:
    telnet google.com 80  

✔ Learned:
- Connected → Network + port working  
- Connection refused → service down / port closed  
- Timeout → firewall or routing issue  

---

### 4️⃣ Learned Packet Analysis using `tcpdump`

Command:
    sudo tcpdump -i eth0 port 80  

✔ Learned:
- Captures live network traffic  
- Shows packet flow (incoming/outgoing)  
- Helps detect:
  - No response from server  
  - Blocked or dropped packets  

💡 Insight:
If packets go out but no response returns → issue likely outside system  

---

### 5️⃣ Learned Network Path Debugging using `mtr`

Command:
    mtr google.com  

✔ Learned:
- Combines ping + traceroute  
- Displays:
  - Network hops  
  - Packet loss  
  - Latency  

💡 Insight:
Useful for identifying slow or failing network paths  

---

### 6️⃣ Practiced Real-World Troubleshooting Scenario

Scenario: Web application is down  

#### Step 1: DNS Check
    dig +short yourapp.com  

- No IP → DNS issue  
- Wrong IP → incorrect configuration  

#### Step 2: Connectivity Check
    telnet yourapp.com 80  

- Timeout → firewall/network issue  
- Refused → service not running  

#### Step 3: Load Balancer Check
- Verified target health  
- Checked listener configuration  

#### Step 4: Firewall Check
- Security groups allow traffic?  
- Network ACL blocking?  

#### Step 5: Server Check
    systemctl status nginx  

- Service running or not  

#### Step 6: Application Logs
    tail -f /var/log/app.log  

- Checked for errors and crashes
