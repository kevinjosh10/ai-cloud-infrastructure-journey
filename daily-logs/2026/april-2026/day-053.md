**Date:** 23 April 2026

---

## 🎯 Objective

Understand how to use boto3 to interact with AWS EC2.  
Learn how to list, filter, and manage EC2 instances programmatically.  
Build a real-world automation script to fetch instance data across all AWS regions.

---

## ✅ What I Completed

### 1️⃣ Learned boto3 EC2 Client Basics
- Created EC2 client using boto3
- Understood how AWS services are accessed via Python
- Learned difference between client-based API and higher-level abstractions

Example:
    ec2 = boto3.client('ec2', region_name='us-east-1')

---

### 2️⃣ Explored describe_instances() Response Structure
- Retrieved EC2 instance data using API
- Understood deeply nested JSON structure
- Learned about Reservations → Instances hierarchy

Structure:
    Reservations → Instances → Instance Details

---

### 3️⃣ Parsed Instance Data from Nested Response
- Iterated through reservations and instances
- Extracted key fields:
  - Instance ID  
  - Instance Type  
  - State  
  - Private IP  
  - Public IP  
  - Tags  

- Used `.get()` to safely access values

---

### 4️⃣ Applied Filters to EC2 Queries
- Filtered instances based on state

Example:
    Filters = [
        {
            'Name': 'instance-state-name',
            'Values': ['running']
        }
    ]

- Learned how filtering reduces unnecessary data processing

---

### 5️⃣ Performed Instance Operations (Start/Stop)
- Used boto3 methods to control EC2 instances

Commands:
    ec2.start_instances(InstanceIds=['i-abc123'])  
    ec2.stop_instances(InstanceIds=['i-abc123'])  

- Understood real-world implications (cost, uptime)

---

### 6️⃣ Implemented Multi-Region Automation
- Used describe_regions() to fetch all AWS regions
- Iterated through each region dynamically
- Queried EC2 instances region by region

Key concept:
- Cloud infrastructure is distributed globally  
- Automation must account for all regions  

---

### 7️⃣ Built ec2_report.py Script
- Created a script to:
  - Loop through all regions  
  - Fetch all EC2 instances  
  - Extract relevant details  
  - Display formatted output  

Output Format:
    Region | Instance ID | Type | State  

- Built a mini infrastructure inventory tool
