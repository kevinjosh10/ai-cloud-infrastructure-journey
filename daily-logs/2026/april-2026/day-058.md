## 🎯 Objective

Understand Python logging, its levels, and its importance in real-world applications.  
Learn how to replace `print()` statements with structured logging and implement logging in cloud-ready systems.

---

## ✅ What I Completed

### 1️⃣ Learned Logging Fundamentals
- Understood logging is used for tracking events in applications  
- Learned difference between `print()` and logging  
- Recognized importance in debugging and production systems  
- Understood logging is essential for cloud and backend systems  

---

### 2️⃣ Understood Logging Levels
- Learned different logging severity levels:
  - DEBUG → Detailed internal information  
  - INFO → General execution updates  
  - WARNING → Non-critical issues  
  - ERROR → Operation failures  
  - CRITICAL → System-level failures  

Example:
    logging.info("Application started")  
    logging.error("Failed to connect to AWS")  

---

### 3️⃣ Configured Basic Logging
- Used `logging.basicConfig()` to initialize logging  
- Set log level and message format  
- Included timestamp, module name, and log level  

Example:
    logging.basicConfig(
        level=logging.INFO,
        format='%(asctime)s - %(name)s - %(levelname)s - %(message)s'
    )

---

### 4️⃣ Learned Logging to File & Console
- Used `FileHandler` to store logs in a file  
- Used `StreamHandler` for console output  
- Combined both for better debugging and monitoring  

Example:
    FileHandler("app.log")  
    StreamHandler()  

---

### 5️⃣ Learned Log Rotation
- Used `RotatingFileHandler` to manage log file size  

Example:
    RotatingFileHandler(maxBytes=10MB, backupCount=5)  

Understood:
- Prevents logs from growing indefinitely  
- Keeps backup log files  
- Important for production systems  

---

### 6️⃣ Built Reusable Logger Module
- Created centralized `logger.py`  
- Used `get_logger(__name__)` pattern  
- Avoided duplicate configurations across scripts  

---

### 7️⃣ Applied Logging to Existing Scripts
- Replaced all `print()` statements  
- Added:
  - INFO logs for workflow steps  
  - DEBUG logs for values  
  - ERROR logs for failures  

- Improved debugging and readability
