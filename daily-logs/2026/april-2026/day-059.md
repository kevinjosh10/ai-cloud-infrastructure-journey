---

### 2️⃣ Understood YAML Syntax

#### Key-Value Pairs
    region: ap-south-1  
    log_level: INFO  

#### Lists
    regions:
      - ap-south-1
      - us-east-1
      - eu-west-1  

#### Nested Objects
    filters:
      instance_state: running
      instance_type: t2.micro  

Rules:
- Indentation-based (spaces only, no tabs)  
- Typically 2 spaces per level  
- No commas or brackets like JSON  

---

### 3️⃣ Installed & Used PyYAML
- Installed library using:
    pip install pyyaml  

- Imported YAML in Python scripts

---

### 4️⃣ Learned Reading YAML Safely

Used:
    yaml.safe_load(file)  

- Loaded YAML config into Python dictionary  
- Avoided using `yaml.load()` due to security risks  
- Understood safe parsing prevents execution of malicious code  

---

### 5️⃣ Learned Writing YAML Files

Used:
    yaml.dump(data, file, default_flow_style=False)  

- Converted Python dictionary → YAML  
- Ensured readable multi-line format  
- Avoided inline compact YAML for better clarity  

---

### 6️⃣ Created Real Config File

Created `config.yaml` for AWS CLI tool:

    regions:
      - ap-south-1
      - us-east-1

    filters:
      instance_state: running
      instance_type: t2.micro

    output:
      format: table

    logging:
      level: INFO  

- Structured configuration logically  
- Grouped related settings  
- Made tool flexible and environment-independent  

---

### 7️⃣ Integrated YAML with CLI Tool

- Loaded config file inside Python script  
- Replaced hardcoded values with config-driven values  

Implemented logic:

    CLI arguments > config.yaml > default values  

- Built function to merge CLI + config values  
- Enabled dynamic configuration

### 8️⃣ Practiced Real-World Behavior
