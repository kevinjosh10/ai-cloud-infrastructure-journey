## ✅ What I Completed

### 1️⃣ Studied the OSI Model Framework

Learned about the **OSI (Open Systems Interconnection) Model**, a conceptual framework used to understand how different networking processes interact to send and receive data.

Key understanding:

- OSI Model has **7 layers**
- Each layer performs a specific function
- Layers work together to enable network communication
- Helps standardize how systems communicate

OSI Layers Overview:

```
1. Physical
2. Data Link
3. Network
4. Transport
5. Session
6. Presentation
7. Application
```

Focused today on **Layers 1–4**, which handle the **core transmission of data**.

---

### 2️⃣ Learned Layer 1 – Physical Layer

The **Physical Layer** is responsible for transmitting raw bits over a physical medium.

Key responsibilities:

- Sending binary data (0s and 1s)
- Converting digital data into electrical or optical signals
- Managing cables, connectors, and physical hardware

Examples of hardware:

- Ethernet cables
- Fiber optic cables
- Network hubs
- Repeaters

Data unit:

**Bits**

---

### 3️⃣ Learned Layer 2 – Data Link Layer

The **Data Link Layer** ensures reliable communication between devices on the same local network.

Key responsibilities:

- Organizing bits into frames
- Error detection
- Managing MAC addresses
- Controlling communication between devices on the same network

Devices operating at this layer:

- Network switches
- Network Interface Cards (NIC)

Address type used:

**MAC Address**

Example:

```
00:1A:2B:3C:4D:5E
```

Data unit:

**Frames**

---

### 4️⃣ Learned Layer 3 – Network Layer

The **Network Layer** is responsible for routing data between different networks.

Key responsibilities:

- Logical addressing using IP addresses
- Determining the best path for data transmission
- Routing packets across networks

Devices operating at this layer:

- Routers
- Layer 3 switches

Address type used:

**IP Address**

Example:

```
192.168.1.1
```

Data unit:

**Packets**

---

### 5️⃣ Learned Layer 4 – Transport Layer

The **Transport Layer** ensures reliable end-to-end communication between systems.

Key responsibilities:

- Segmenting data
- Managing flow control
- Error recovery
- Ensuring correct data delivery

Important protocols:

**TCP – Transmission Control Protocol**

- Reliable
- Connection-oriented
- Used for web browsing, email, and file transfers

**UDP – User Datagram Protocol**

- Faster but less reliable
- Connectionless
- Used for streaming, gaming, and DNS

Data unit:

**Segments**
