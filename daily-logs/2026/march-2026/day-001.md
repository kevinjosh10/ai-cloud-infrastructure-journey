# Day 001 – AWS CLI & IAM Foundation

**Date:** 04 March 2026  
**Duration:** ~2 Hours  
**Focus:** AWS Fundamentals & Environment Setup  

---

## 🎯 Objective

Set up a secure AWS environment and establish programmatic access using AWS CLI.  
Build strong IAM and identity fundamentals before creating infrastructure.

---

## ✅ What I Completed

### 1️⃣ Secured AWS Access
- Used IAM user instead of root account
- Reviewed attached policies
- Understood risks of access key leakage
- Confirmed `AdministratorAccess` permissions

---

### 2️⃣ Installed & Configured AWS CLI
- Installed AWS CLI v2 on Windows
- Ran `aws configure`
- Set default region to `ap-south-1`

Verified configuration using:

```powershell
aws --version
aws configure list
```

---

### 3️⃣ Verified Identity Using STS

Command used:

```powershell
aws sts get-caller-identity
```

Learned:
- How AWS identifies API callers
- Difference between IAM User and Role
- Importance of identity verification

---

### 4️⃣ Tested S3 Access

Command used:

```powershell
aws s3 ls
```

Confirmed:
- CLI authentication works
- Permissions are active
- Programmatic AWS access established

---

### 5️⃣ Inspected IAM Configuration

Commands used:

```powershell
aws iam list-users
aws iam list-attached-user-policies --user-name kevin-admin1
```

Learned:
- Managed vs Inline policies
- `AdministratorAccess` provides full control
- Principle of least privilege

---

## 🧠 Key Concepts Learned

- AWS Region vs Availability Zone  
- IAM User vs IAM Role  
- STS (Security Token Service)  
- Access key security risks  
- CLI-based infrastructure control  
- Credential storage in `.aws` directory
