## ✅ What I Completed

### 1️⃣ Studied OSI Layer 5 – Session Layer

The **Session Layer** is responsible for establishing and managing communication sessions between two devices.

Key responsibilities:

- Establishing communication sessions
- Maintaining active sessions
- Terminating sessions when communication ends

Example:

When logging into a website, a **session is created** that keeps the user authenticated while interacting with the service.

This layer ensures communication remains synchronized during data exchange.

Data unit:

**Data**

---

### 2️⃣ Studied OSI Layer 6 – Presentation Layer

The **Presentation Layer** acts as the translator between different systems so that data can be understood by both sides.

Key responsibilities:

- Data translation
- Encryption and decryption
- Data compression

Example:

Secure websites using **HTTPS** rely on encryption technologies such as **SSL/TLS** to protect communication between the browser and server.

This layer ensures that data is correctly formatted and secure during transmission.

Data unit:

**Data**

---

### 3️⃣ Studied OSI Layer 7 – Application Layer

The **Application Layer** is the closest layer to the user and provides network services to applications.

This is the layer where **web browsing, email communication, and file transfers occur**.

Common protocols studied:

**HTTP – Hypertext Transfer Protocol**

- Used to load websites

**HTTPS – Hypertext Transfer Protocol Secure**

- Encrypted version of HTTP

**FTP – File Transfer Protocol**

- Used for transferring files between systems

**DNS – Domain Name System**

- Converts domain names into IP addresses

Example:

```
google.com → 142.250.x.x
```

**SMTP – Simple Mail Transfer Protocol**

- Used for sending emails

This layer allows user-facing applications to interact with the network.

---

### 4️⃣ Learned the TCP/IP Model

The **TCP/IP model** is the networking architecture that powers the modern internet.

Unlike the OSI model, it consists of **4 layers**.

TCP/IP Layers:

```
1. Application
2. Transport
3. Internet
4. Network Access
```

Each layer combines functions from multiple OSI layers.

This model is used in **real-world networking and internet communication**.

---

### 5️⃣ Compared OSI Model vs TCP/IP Model

A key learning was understanding how the two models relate to each other.

| OSI Model | TCP/IP Model |
|------|------|
| Application | Application |
| Presentation | Application |
| Session | Application |
| Transport | Transport |
| Network | Internet |
| Data Link | Network Access |
| Physical | Network Access |

Key insight:

- **OSI model is conceptual**
- **TCP/IP model is practical and used in real networks**

### 6️⃣ Learned IP Addressing Basics

Every device connected to a network requires an **IP address**.

Studied the **IPv4 address format**.

Example format:

```
x.x.x.x
```

Example IP address:

```
192.168.1.1
```

Each section ranges from **0–255**.

IP addresses allow devices to identify and communicate with each other across networks.

---

### 7️⃣ Understood Private vs Public IP Addresses

Private IP addresses are used within local networks and cannot be accessed directly from the internet.

Private IP ranges:

```
10.0.0.0 – 10.255.255.255
172.16.0.0 – 172.31.255.255
192.168.0.0 – 192.168.255.255
```

Example private IP:

```
192.168.1.10
```

Public IP addresses are globally reachable and used to identify devices on the internet.

Example public IP:

```
8.8.8.8
```

(Google Public DNS)

### 8️⃣ Practiced Network Diagnostic Commands

Used basic networking commands to observe how systems communicate across the internet.

**Ping**

Command used:

```
ping google.com
```

Purpose:

- Test network connectivity
- Measure latency
- Check if a server is reachable

Ping works using the **ICMP protocol**.

---

### 9️⃣ Used Traceroute to Observe Network Path

Traceroute shows the path data packets take to reach a destination server.

Windows command used:

```
tracert google.com
```

This command displays **each router (hop)** the packet travels through.

Example path:

```
Laptop
↓
Home Router
↓
Internet Service Provider
↓
Internet Backbone
↓
Destination Server
```

Traceroute is commonly used by network engineers and cloud engineers for troubleshooting connectivity issues.
