# AWS CLI Resource Inspection

The AWS CLI allows engineers to quickly inspect resources within an AWS account without navigating the console.

### Listing S3 Buckets

This command lists all Amazon S3 buckets associated with the AWS account.

```bash
aws s3 ls
```

This helps quickly verify available storage resources.

---

### Viewing EC2 Instances

The following command retrieves detailed information about EC2 instances in the configured AWS region.

```bash
aws ec2 describe-instances
```

This command returns infrastructure details in JSON format, including instance IDs, states, networking details, and more.

---

# Amazon S3 Operations

Amazon S3 is AWS’s **object storage service**, widely used for storing application assets, logs, backups, and datasets.

Using the CLI, engineers can manage S3 buckets and objects efficiently.

---

## Creating an S3 Bucket

```bash
aws s3 mb s3://kevin-test-bucket-156
```

One important observation learned today:

**S3 bucket names must be globally unique across all AWS users.**

If a bucket name already exists anywhere in AWS, the bucket creation request will fail.

---

## Uploading a File to S3

```bash
aws s3 cp test.txt s3://kevin-test-bucket-156
```

This command uploads a file from the local system to the specified S3 bucket.

---

## Listing Files in a Bucket

```bash
aws s3 ls s3://kevin-test-bucket-156
```

Displays all objects stored in the bucket.

---

## Deleting a File from a Bucket

```bash
aws s3 rm s3://kevin-test-bucket-156/test.txt
```

Removes the specified file from the bucket.

# AWS CLI Profiles

AWS CLI profiles allow multiple AWS credentials to be stored and used on the same machine.

This is extremely useful when working with different environments such as:

- Development
- Testing
- Production
- Multiple AWS accounts

---

## Creating a CLI Profile

A new profile can be created using:

```bash
aws configure --profile dev
```

During configuration, AWS CLI asks for the following information:

- AWS Access Key ID
- AWS Secret Access Key
- Default Region
- Default Output Format

Example region used:

```
ap-south-1
```

This corresponds to the **Asia Pacific (Mumbai)** AWS region.

---

## Using a CLI Profile

Commands can be executed using a specific profile.

```bash
aws s3 ls --profile dev
```

This ensures the command uses credentials stored in the `dev` profile.

---

## Listing Configured Profiles

To view all configured CLI profiles:

```bash
aws configure list-profiles
```

# AWS CLI Output Formatting

AWS CLI supports multiple output formats which are useful for readability and automation.

---

## JSON Output

```bash
aws ec2 describe-instances --output json
```

JSON format is ideal for scripting and automation workflows.

---

## Table Output

```bash
aws ec2 describe-instances --output table
```

Table format makes the output easier for humans to read.

---

## Filtering Results Using Query

The `--query` option allows filtering specific fields from AWS API responses.

```bash
aws ec2 describe-instances --query 'Reservations[*].Instances[*].InstanceId'
```

This is extremely useful when building automation scripts or extracting specific data from AWS responses.
