### 3️⃣ Created a Python Virtual Environment

A **virtual environment** isolates project dependencies so that libraries installed for one project do not affect others.

Command used:

```
python -m venv venv
```

This created the following structure:

```
aws-python-project
│
└── venv
```

The `venv` directory contains a **separate Python runtime and package manager for the project**.

---

### 4️⃣ Activated the Virtual Environment

Activated the virtual environment using PowerShell.

Command used:

```
venv\Scripts\activate
```

Terminal output changed to:

```
(venv) C:\Users\ADMIN\OneDrive\Desktop\aws-python-project>
```

The `(venv)` prefix confirms that the **virtual environment is active**.

This ensures all installed packages remain **isolated within this project**.

### 7️⃣ Learned the Purpose of Key Python Libraries

**boto3 – AWS SDK for Python**

Allows Python programs to interact with AWS services.

Examples:

- Create EC2 instances
- Upload files to S3
- Manage IAM roles
- Automate cloud infrastructure

---

**requests – HTTP Client Library**

Used to make API calls and communicate with web services.

Example usage:

```
import requests

response = requests.get("https://api.github.com")
print(response.json())
```

This is commonly used when interacting with **external APIs and cloud services**.

---

**pyyaml – YAML Processing Library**

Used to read and write **YAML configuration files**.

YAML is widely used in modern infrastructure tools such as:

- Kubernetes
- CI/CD pipelines
- Infrastructure configuration files

Example import:

```
import yaml
```

### 8️⃣ Explored the AWS Management Console

Spent time navigating the **AWS Console** to understand core cloud services.

The AWS Console is a web interface used to **create, configure, and monitor cloud infrastructure**.

Services explored included:

---

**EC2 – Elastic Compute Cloud**

Provides scalable virtual servers in the cloud.

Common uses:

- Hosting applications
- Backend services
- Running containers
- AI workloads

Explored sections:

- Instances
- AMIs
- Security Groups
- Key Pairs

---

**S3 – Simple Storage Service**

Object storage service used for storing files and data.

Typical use cases:

- Static website hosting
- Backup storage
- Log storage
- AI datasets

Explored:

- Buckets
- Objects
- Permissions
- Storage classes

---

**IAM – Identity and Access Management**

Used to control **who can access AWS resources and what actions they can perform**.

Explored:

- Users
- Roles
- Groups
- Policies

IAM is essential for maintaining **security and access control in cloud environments**.

---

**VPC – Virtual Private Cloud**

Provides a **private network inside AWS** where resources can securely communicate.

Explored:

- VPC configuration
- Subnets
- Route tables
- Internet gateways

This forms the networking foundation of cloud infrastructure.

---

**CloudWatch – Monitoring & Observability**

AWS monitoring service used to observe system performance and logs.

Explored features:

- Metrics
- Logs
- Alarms
- Dashboards

CloudWatch helps monitor:

- CPU usage
- Network traffic
- Application logs
- System health

---

### 9️⃣ Launched a Free Tier EC2 Instance

Performed a practical exercise by launching a virtual server.

Steps followed:

1. Open AWS Console
2. Navigate to EC2
3. Click **Launch Instance**
4. Select **Amazon Linux AMI**
5. Choose **t2.micro (Free Tier eligible)**
6. Create a **Key Pair**
7. Launch the instance

Instance status changed to:

```
Running
```

This represents a **live virtual machine running in the cloud**.

---

### 🔟 Terminated the EC2 Instance

After testing, terminated the instance to avoid unnecessary charges.

Steps followed:

1. Select instance
2. Click **Instance State**
3. Choose **Terminate Instance**

Instance status became:

```
Terminated
```

This demonstrates the **lifecycle management of cloud compute resources**.
