# 🌐 OSI Model Review

The **OSI (Open Systems Interconnection) Model** explains how data travels across networks using a **7-layer architecture**.

Each layer has a specific responsibility in ensuring that data can move reliably from one system to another.

---

## 📊 OSI Layers Overview

| Layer | Name | Responsibility |
|-----|-----|-----|
| 7 | Application | Interfaces with user applications (HTTP, DNS, FTP) |
| 6 | Presentation | Data formatting, encryption, compression |
| 5 | Session | Manages communication sessions |
| 4 | Transport | Reliable data transfer (TCP / UDP) |
| 3 | Network | Packet routing using IP addresses |
| 2 | Data Link | MAC addressing and switching |
| 1 | Physical | Transmission of raw bits through cables or signals |

Understanding these layers helps engineers **debug networking issues and design scalable systems.**

---

# 🌍 TCP/IP Model

While OSI is a conceptual model, the **TCP/IP model is what the internet actually uses.**

It simplifies networking into **4 layers**.

| TCP/IP Layer | Related OSI Layers |
|--------------|-------------------|
| Application | OSI Layers 5–7 |
| Transport | OSI Layer 4 |
| Internet | OSI Layer 3 |
| Network Access | OSI Layers 1–2 |

This model powers **modern internet communication and cloud infrastructure networking.**

# 🔗 TCP vs UDP

Transport protocols define **how data is delivered between systems.**

---

## 📡 TCP (Transmission Control Protocol)

TCP provides **reliable communication**.

Key characteristics:

- Connection-oriented  
- Ensures packet delivery  
- Maintains correct order of packets  

TCP establishes connections using the **3-way handshake**:

1️⃣ SYN → Client requests connection  
2️⃣ SYN-ACK → Server acknowledges request  
3️⃣ ACK → Client confirms connection

Common use cases:

- Web browsing (HTTP / HTTPS)  
- File transfers  
- Email systems  

---

## ⚡ UDP (User Datagram Protocol)

UDP prioritizes **speed over reliability**.

Key characteristics:

- Connectionless protocol  
- Faster transmission  
- No guarantee of packet delivery  

Common use cases:

- Live video streaming  
- Online gaming  
- DNS queries  
- Voice communication

# 🌍 IP Addressing Basics

Every device on the internet requires an **IP address** to communicate.

An IPv4 address follows the format:

```bash
x.x.x.x
```

Example:

```bash
192.168.1.1
```

---

## 🏠 Private IP Address Ranges

Private IP addresses are used **within internal networks** and are not accessible directly from the internet.

| Range | Example |
|------|------|
| 10.0.0.0 – 10.255.255.255 | 10.0.0.5 |
| 172.16.0.0 – 172.31.255.255 | 172.16.10.1 |
| 192.168.0.0 – 192.168.255.255 | 192.168.1.25 |

Routers use **NAT (Network Address Translation)** to convert private IPs into public ones for internet communication.

# 💻 Network Diagnostics in the Terminal

Cloud engineers frequently use terminal tools to **diagnose connectivity and routing issues.**

---

## 📡 Ping Command

Tests whether a host is reachable on the network.

```bash
ping google.com
```

If the server responds, packets are successfully reaching the destination.

---

## 🛰 Traceroute Command

Displays the route packets take across the internet.

Mac / Linux:

```bash
traceroute google.com
```

Windows:

```bash
tracert google.com
```

This command reveals **every router hop between the local machine and the destination server.**
