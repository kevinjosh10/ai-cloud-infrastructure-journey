# 🧭 Linux File System Hierarchy

Unlike Windows which uses multiple drives (`C:\`, `D:\`), Linux organizes everything under a **single root directory**.

```
/
```

All files, folders, applications, and configurations exist somewhere under this root.

Example simplified structure:

```
/
├── bin
├── etc
├── home
├── usr
├── var
├── tmp
├── opt
├── sbin
└── mnt
```

Each directory has a specific purpose within the system.

---

# 📂 Important Linux Directories

Understanding these directories is essential for navigating Linux servers.

---

## 🏠 `/home` — User Directories

Stores personal user files and directories.

Example:

```
/home/kevin
/home/user
```

Comparable to:

```
C:\Users\Kevin
```

This is where user documents, downloads, and personal projects are stored.

---

## ⚙ `/etc` — System Configuration

Contains configuration files for the operating system and installed services.

Examples:

```
/etc/passwd
/etc/ssh
/etc/network
```

System administrators edit files here to configure system behavior.

---

## 📊 `/var` — Variable Data

Stores data that changes frequently while the system runs.

Examples include:

```
/var/log
/var/cache
/var/spool
```

---

## 📜 `/var/log` — System Logs

Stores log files generated by the system and applications.

Examples:

```
syslog
auth.log
kern.log
```

Logs are critical when diagnosing problems on servers.

---

## 📦 `/usr` — User Applications & Libraries

Contains installed programs, utilities, and libraries used by the system.

Examples:

```
/usr/bin
/usr/lib
/usr/share
```

Most system software resides here.

---

## 🧰 `/bin` — Essential Commands

Contains fundamental commands required for system operation.

Examples:

```
ls
cp
mv
cat
```

These commands are available even in minimal environments.

---

## 🔧 `/sbin` — System Administration Commands

Contains commands mainly used for system administration tasks.

Examples:

```
reboot
shutdown
fdisk
```

---

## 🗂 `/tmp` — Temporary Files

Applications store temporary files here while running.

These files may be cleared automatically by the system.

---

## 📥 `/opt` — Optional Software

Used for installing third-party applications.

Example:

```
/opt/docker
/opt/google
```

---

## 💾 `/mnt` — Mount Points

Used when attaching external storage or filesystems.

Examples include:

- USB drives
- External disks
- Network storage

---

# 🖥 Core Linux Navigation Commands

The Linux terminal allows engineers to navigate the filesystem quickly.

---

## 📍 Print Current Directory

```
pwd
```

Displays the current working directory.

Example output:

```
/home/kevin
```

---

## 📂 Listing Files and Directories

```
ls
```

Lists all files and directories in the current location.

---

### Detailed File View

```
ls -l
```

Shows detailed information such as permissions, owner, and file size.

---

### Show Hidden Files

```
ls -a
```

Displays hidden files (files starting with `.`).

---

## 🔁 Changing Directories

Move into another directory:

```
cd Documents
```

---

### Move Up One Directory

```
cd ..
```

---

### Go to Home Directory

```
cd ~
```

---

### Go to Root Directory

```
cd /
```

---

# 📍 Absolute vs Relative Paths

Understanding paths is essential when navigating Linux systems.

---

## Absolute Path

An **absolute path** starts from the root directory (`/`).

Example:

```
/home/kevin/documents
```

No matter where you are in the system, this path always leads to the same location.

---

## Relative Path

A **relative path** is based on the current directory.

Example:

```
../documents
```

Meaning:

- `..` → move up one directory  
- then enter `documents`

---

### Example Scenario

Current directory:

```
/home/kevin/projects
```

Command:

```
cd ../documents
```

Result:

```
/home/kevin/documents
```

# 📝 Key Learnings

- 🐧 Linux organizes all files under a single root directory (`/`).  
- 📂 System directories each serve specific roles such as configuration, logs, and applications.  
- 🧭 Commands like `pwd`, `ls`, and `cd` are essential for navigating Linux systems.  
- 📍 Absolute paths start from root, while relative paths depend on the current location.  
- 📜 System logs stored in `/var/log` are critical for debugging server issues.
