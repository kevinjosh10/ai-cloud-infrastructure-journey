# Day 009 — Linux File & Directory Operations 🗂️

**Date:** 11 March 2026  
**Duration:** ~2 Hours  
**Focus:** Managing Files & Directories in Linux
  
---

## 🎯 Objective

Understand how to **create, copy, move, rename, and delete files and directories** in Linux. These operations are fundamental for working with servers, managing project files, and handling logs in real-world cloud environments.

---

## 📁 Creating Directories

### `mkdir` — Make Directory

Used to create new folders.

```bash
mkdir folder_name
```

Example:

```bash
mkdir projects
```

Create multiple directories:

```bash
mkdir folder1 folder2 folder3
```

Create nested directories:

```bash
mkdir -p cloud/aws/projects
```

---

## 🗑️ Deleting Files & Directories

### `rm` — Remove File

```bash
rm filename
```

Example:

```bash
rm notes.txt
```

⚠️ Files deleted with `rm` cannot be recovered easily.

---

### `rm -r` — Remove Directory Recursively

```bash
rm -r foldername
```

Example:

```bash
rm -r old_project
```

Removes the directory **and everything inside it**.

---

### `rm -rf` — Force Delete (Use Carefully)

```bash
rm -rf foldername
```

- `-r` → recursive  
- `-f` → force delete

Commonly used when cleaning environments or removing large directories.

---

## 📦 Copying Files & Directories

### `cp` — Copy File

```bash
cp source destination
```

Example:

```bash
cp notes.txt backup.txt
```

Copy file to a folder:

```bash
cp notes.txt documents/
```

---

### Copy Directory

```bash
cp -r project backup_project
```

`-r` copies directories recursively.

---

## 🚚 Moving & Renaming Files

### `mv` — Move File

```bash
mv file.txt folder/
```

Moves the file into another directory.

---

### Rename File

```bash
mv oldname.txt newname.txt
```

Example:

```bash
mv project1 project_final
```

---

## 📄 Creating Files

### `touch` — Create Empty File

```bash
touch filename
```

Example:

```bash
touch notes.txt
```

Create multiple files:

```bash
touch file1.txt file2.txt file3.txt
```

---

## 📖 Viewing File Content

### `cat` — Display File Content

```bash
cat filename
```

Example:

```bash
cat notes.txt
```

Used to quickly view file contents in the terminal.

---

## 📜 Reading Large Files

### `less` — Scroll Through Files

```bash
less filename
```

Example:

```bash
less server.log
```

Controls:

```
↑ ↓   Scroll
Space Next page
q     Quit
```

This command is widely used for **reading server logs**.

---

## ⭐ Wildcards

Wildcards allow operations on **multiple files at once**.

---

### `*` — Match Multiple Files

List all `.txt` files:

```bash
ls *.txt
```

Delete files starting with "file":

```bash
rm file*
```

Examples:

```
file1.txt
file2.txt
file_backup.txt
```

## ☁️ Cloud Engineering Relevance

File and directory operations are essential when working with:

- **AWS EC2 instances**
- **Server configuration files**
- **Application deployments**
- **System logs**
- **Backup and file management**

Example real-world workflow:

```bash
mkdir my-app
cd my-app
touch config.yaml
cp config.yaml config_backup.yaml
less server.log
rm *.tmp
```

These commands are part of the **daily toolkit of cloud engineers and DevOps professionals**.
