## 📚 Key Concepts Learned

### 1️⃣ Linux Permission Structure

Every file in Linux has permissions assigned to three categories of users:

| Category | Meaning |
|--------|--------|
| Owner | The user who owns the file |
| Group | Users belonging to the file’s group |
| Others | Everyone else |

Permissions are represented using:

```
rwxr-xr--
```

Breakdown:

```
Owner   Group   Others
rwx     r-x     r--
```

| Symbol | Meaning | Value |
|------|------|------|
| r | Read | 4 |
| w | Write | 2 |
| x | Execute | 1 |

---

### 2️⃣ Viewing Permissions

Command:

```bash
ls -la
```

Example output:

```
-rwxr-xr-- 1 kevin staff 120 Mar 11 script.sh
```

Explanation:

| Section | Meaning |
|------|------|
| - | File type |
| rwx | Owner permissions |
| r-x | Group permissions |
| r-- | Others permissions |
| kevin | Owner |
| staff | Group |

---

### 3️⃣ Numeric Permission System

Permissions can also be represented using numbers.

| Permission | Calculation | Number |
|-----------|------------|--------|
| rwx | 4 + 2 + 1 | 7 |
| rw- | 4 + 2 | 6 |
| r-x | 4 + 1 | 5 |
| r-- | 4 | 4 |

Example:

```
rwxr-xr-x
```

Split into three parts:

```
rwx | r-x | r-x
 7  |  5  |  5
```

Numeric form:

```
755
```

---

### 4️⃣ Changing Permissions with chmod

`chmod` means **change mode**.

Syntax:

```bash
chmod permissions filename
```

Example:

```bash
chmod 755 script.sh
```

This means:

| User | Permission |
|------|-----------|
| Owner | rwx |
| Group | r-x |
| Others | r-x |

Common permission values:

| Number | Meaning |
|------|------|
| 777 | Full access for everyone (not secure) |
| 755 | Owner full, others read + execute |
| 644 | Owner read/write, others read |
| 600 | Private file |

Example:

```bash
chmod 644 notes.txt
```

---

### 5️⃣ Creating an Executable Script

Step 1 — Create script file

```bash
nano script.sh
```

Add:

```bash
#!/bin/bash
echo "Hello Linux"
```

---

Step 2 — Make script executable

```bash
chmod 755 script.sh
```

---

Step 3 — Verify permissions

```bash
ls -la
```

Example:

```
-rwxr-xr-x script.sh
```

---

Step 4 — Run the script

```bash
./script.sh
```

Output:

```
Hello Linux
```

---

### 6️⃣ Changing File Ownership with chown

`chown` means **change owner**.

Syntax:

```bash
chown user:group file
```

Example:

```bash
chown kevin:staff file.txt
```

Verify ownership:

```bash
ls -la
```

Example:

```
-rw-r--r-- 1 kevin staff file.txt
```

---

### 7️⃣ Special Permissions (setuid & setgid)

These are advanced Linux permission mechanisms.

#### setuid

Allows a file to run with the **owner’s privileges** instead of the user's.

Example:

```
-rwsr-xr-x
```

The `s` indicates **setuid**.

A common example:

```
/usr/bin/passwd
```

This allows users to change their passwords even though the password file belongs to `root`.

---

#### setgid

When applied to directories, new files inherit the **group of the directory**.

Example:

```
drwxr-sr-x
```

This is useful for **shared team directories**.

## ☁️ Why This Matters for Cloud Engineers

Linux permissions are critical for:

- Server security
- SSH key protection
- Script execution
- DevOps automation
- Docker container permissions
- Deployment pipelines

Example:

SSH keys require strict permissions.

```bash
chmod 400 key.pem
```

Without correct permissions, SSH will refuse to connect.
