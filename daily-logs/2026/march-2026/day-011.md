---

## 📚 Key Concepts Learned

### 1️⃣ `grep` — Searching Text Patterns

`grep` is used to search for specific patterns inside files.

Example:

```bash
grep "ERROR" server.log
```

This command prints all lines containing **ERROR**.

Useful for quickly identifying issues inside logs.

---

### 2️⃣ Important `grep` Options

`grep -i` → case-insensitive search

```bash
grep -i "error" server.log
```

Matches:

```
ERROR
Error
error
```

---

`grep -n` → show line numbers

```bash
grep -n "ERROR" server.log
```

Example output:

```
3:ERROR Database connection failed
5:ERROR Timeout occurred
```

---

`grep -r` → recursive search in directories

```bash
grep -r "ERROR" logs/
```

Searches for the pattern across **multiple files inside a folder**.

---

### 3️⃣ `head` — View Beginning of Files

Displays the first **10 lines** of a file.

```bash
head server.log
```

Specify number of lines:

```bash
head -n 5 server.log
```

Useful for quickly checking **file headers or initial log entries**.

---

### 4️⃣ `tail` — View End of Files

Displays the last **10 lines**.

```bash
tail server.log
```

Specify number of lines:

```bash
tail -n 5 server.log
```

Useful for checking the **most recent log entries**.

---

### 🔥 `tail -f` — Real-Time Log Monitoring

This command follows a log file as it updates.

```bash
tail -f server.log
```

Used in real systems to **watch logs live while an application is running**.

Example usage:

```
tail -f /var/log/syslog
```

---

### 5️⃣ `wc` — Counting Text

Counts lines, words, and characters.

Example:

```bash
wc server.log
```

Common usage:

```bash
wc -l server.log
```

Counts **number of lines** in the file.

---

### 6️⃣ `sort` — Sorting Text

Sorts lines alphabetically.

```bash
sort names.txt
```

Example input:

```
Zara
Adam
John
```

Output:

```
Adam
John
Zara
```

---

### 7️⃣ `uniq` — Removing Duplicate Lines

Removes duplicate lines from sorted text.

Example:

```bash
sort server.log | uniq
```

Important: `uniq` works best when duplicates are **adjacent**, which is why it is often combined with `sort`.

---

### 8️⃣ Pipes `|` — Combining Commands

Pipes allow multiple commands to work together.

Example:

```bash
grep "ERROR" server.log | wc -l
```

Steps:

1. `grep` finds all ERROR lines  
2. `wc -l` counts them  

Result: **Total number of errors in the log file**

```bash

```

View last lines:

```bash
tail server.log
```


```bash
sort server.log | uniq
```
