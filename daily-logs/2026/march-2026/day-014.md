# 1️⃣ Navigation Commands

### `pwd`
Prints the current working directory.

Example:
```bash
pwd
```

---

### `ls`
Lists files and directories inside the current directory.

Example:
```bash
ls
```

---

### `ls -la`
Lists all files including hidden files and shows detailed information such as permissions, owner, size, and modification time.

Example:
```bash
ls -la
```

---

### `cd`
Changes the current directory.

Example:
```bash
cd /home
```

---

### `cd ..`
Moves one directory up.

Example:
```bash
cd ..
```

---

### `cd ~`
Moves to the user's home directory.

Example:
```bash
cd ~
```

---

# 2️⃣ File & Directory Operations

### `mkdir`
Creates a new directory.

Example:
```bash
mkdir project1
```

---

### `touch`
Creates an empty file.

Example:
```bash
touch file.txt
```

---

### `cp`
Copies files from one location to another.

Example:
```bash
cp file.txt backup.txt
```

---

### `mv`
Moves or renames files.

Example (rename):
```bash
mv file.txt newfile.txt
```

Example (move):
```bash
mv file.txt folder/
```

---

### `rm`
Deletes a file.

Example:
```bash
rm file.txt
```

---

### `rm -r`
Deletes a directory and everything inside it.

Example:
```bash
rm -r project1
```

# 4️⃣ Wildcards

### `*`
Matches any number of characters.

Example:

```bash
ls *.txt
```

---

Example:

# 5️⃣ File Permissions

Linux permissions are structured as:

```
rwxr-xr--
```

Meaning:

| Section | Represents |
|------|------|
| First | Owner |
| Second | Group |
| Third | Others |

Permission values:

| Permission | Value |
|------|------|
| Read | 4 |
| Write | 2 |
| Execute | 1 |

---

### `chmod`
Changes file permissions.

Example:
```bash
chmod 755 script.sh
```

Meaning:

- Owner → read, write, execute
- Group → read, execute
- Others → read, execute

---

### `chown`
Changes file ownership.

Example:
```bash
chown kevin:staff file.txt
```

# 6️⃣ Text Processing Commands

### `grep`
Searches for text patterns inside files.

Example:
```bash
grep "error" logfile.txt
```

---

### `grep -i`
Case-insensitive search.

Example:
```bash
grep -i error logfile.txt
```

---

### `grep -n`
Shows line numbers of matching results.

Example:
```bash
grep -n error logfile.txt
```

---

### `wc -l`
Counts number of lines.

Example:
```bash
wc -l logfile.txt
```

---

### `sort`
Sorts lines alphabetically.

Example:
```bash
sort names.txt
```

---

### `uniq`
Removes duplicate lines.

Example:
```bash
uniq names.txt
```

---

# 7️⃣ Pipes

### `|`
Pipes send the output of one command as input to another.

Example:

```bash
ls -la | grep ".txt"
```

Example:

```bash
ls -la | grep ".txt" | wc -l
```

This lists `.txt` files and counts them.

---

# 8️⃣ Redirection

### `>`
Redirects output and overwrites a file.

Example:
```bash
echo "hello" > file.txt
```

---

### `>>`
Appends output to a file.

Example:
```bash
echo "world" >> file.txt
```

---

### `<`
Uses file content as input.

Example:
```bash
cat < file.txt
```

---

### `2>`
Redirects errors to a file.

Example:
```bash
ls wrongfolder 2> error.log
```

---

---

### `$HOME`

---

# 🔎 Searching Commands

### `find`
Searches for files within directories.

Example:
```bash
find . -name "*.txt"
```

---

### `-type f`
Search for files only.

Example:
```bash
find . -type f
```

---

### `-type d`
Search for directories only.

Example:
```bash
find . -type d
```

---

### `locate`
Fast file search using an indexed database.

Example:
```bash
locate file.txt
```

---

### `which`
Shows where a command is located.

Example:
```bash
which python
```

---

# ⚙️ Command Combination Examples

Find errors in logs:

```bash
grep -i error server.log
```

Count errors:

```bash
grep -i error server.log | wc -l
```

Monitor logs live:

```bash
tail -f server.log
```

Find running process:

```bash
ps aux | grep nginx
```

Delete multiple files:

```bash
find . -name "*.log" | xargs rm
```
