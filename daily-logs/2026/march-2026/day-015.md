### 💻 Understanding How Linux Runs Programs

Today’s focus was on **process management in Linux** — learning how the operating system runs, monitors, and controls programs.

Every application running on a Linux system — whether it's a **web server, AI model, database, or background service** — runs as a **process**.

For **Cloud Engineers, DevOps Engineers, and Site Reliability Engineers (SREs)**, understanding processes is essential for **debugging slow servers, identifying resource-heavy programs, and managing running services**.

---

## 📊 Viewing Running Processes with `ps`

The `ps` command displays **information about active processes**.

Example:

```bash
ps aux
```

This lists **all running processes with detailed information**.

Example output columns:

| Column | Meaning |
|------|------|
| USER | User who started the process |
| PID | Process ID |
| %CPU | CPU usage |
| %MEM | Memory usage |
| COMMAND | Program being executed |

Example for engineers:

```bash
ps aux | grep python
```

This searches for **all running Python processes**, which is useful when debugging servers or scripts.

---

## 📈 Real-Time System Monitoring with `top`

The `top` command shows **live system activity and running processes**.

Example:

```bash
top
```

It displays:

- CPU usage  
- Memory usage  
- Running processes  
- System load  

Useful keys inside `top`:

| Key | Action |
|----|----|
| q | Quit |
| k | Kill a process |
| P | Sort by CPU usage |
| M | Sort by memory usage |

This tool is commonly used when **servers become slow or overloaded**.

---

## ⚡ Advanced Monitoring with `htop`

`htop` is an improved and more user-friendly version of `top`.

Install:

```bash
sudo apt install htop
```

Run:

```bash
htop
```

Advantages:

- Colorful visual interface  
- Easier navigation  
- Scrollable process list  
- Ability to kill processes interactively  

Exit with:

```
q
```

---

## 🛑 Stopping Processes with `kill`

Every running process has a **PID (Process ID)**.

To stop a process:

```bash
kill PID
```

This sends the **SIGTERM signal**, which asks the program to terminate gracefully.

Example:

```bash
kill 2456
```

---

## 💥 Force Stopping Processes

Sometimes programs freeze or refuse to stop.

In those cases, a **force kill** can be used:

```bash
kill -9 PID
```

This sends the **SIGKILL signal**, immediately terminating the process.

| Command | Signal | Purpose |
|------|------|------|
| kill | SIGTERM | Graceful shutdown |
| kill -9 | SIGKILL | Force termination |

This is commonly used when **applications become unresponsive on servers**.

---

## 🔄 Background and Foreground Jobs

Linux allows processes to run in the **background**, letting the terminal remain usable.

Example long-running command:

```bash
sleep 1000
```

This command pauses for 1000 seconds and occupies the terminal.

---

### ⏸ Pausing a Process

Press:

```
Ctrl + Z
```

This suspends the process.

Example output:

```
[1]+ Stopped sleep 1000
```

---

### ▶ Sending a Job to the Background

```bash
bg
```

The job now runs in the background.

---

### 📋 Listing Background Jobs

```bash
jobs
```

Example output:

```
[1]+ Running sleep 1000 &
```

---

### 🔙 Bringing a Job Back to Foreground

```bash
fg
```

This returns the process to the active terminal session.

## 🚀 Key Takeaways

- Linux runs programs as **processes**, each identified by a unique **PID**.  
- `ps` allows engineers to **view running processes**.  
- `top` and `htop` provide **real-time system monitoring**.  
- `kill` and `kill -9` are used to **stop processes**.  
- `bg`, `fg`, `jobs`, and `Ctrl + Z` help manage **background and foreground tasks**.

These commands are essential for **managing servers, debugging applications, and monitoring cloud infrastructure**.
