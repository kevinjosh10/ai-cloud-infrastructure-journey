## 📦 What is a Package Manager?

A package manager is a tool that:
- Installs software
- Updates software
- Removes software
- Automatically handles dependencies

Think of it as an **App Store for Linux systems**, but for servers.

---

## 🐧 APT (Ubuntu / Debian)

### 🔹 Update Package List
    sudo apt update

### 🔹 Upgrade Installed Packages
    sudo apt upgrade

### 🔹 Install a Package
    sudo apt install nginx

### 🔹 Search for Packages
    apt search nginx

### 🔹 View Package Details
    apt show nginx

### 🔹 Remove a Package
    sudo apt remove nginx

---

## 🧰 YUM / DNF (CentOS / RHEL)

### 🔹 Install Package
    sudo yum install nginx

or (newer systems):
    sudo dnf install nginx

### 🔹 List Installed Packages
    yum list installed

---

## 📋 Package Listing Tools

### 🔹 Debian-based Systems
    dpkg -l

### 🔹 RedHat-based Systems
    rpm -qa

## ⚙️ Hands-On Practice — Nginx Setup

### ✅ Step 1: Update & Install Nginx
    sudo apt update
    sudo apt install nginx -y

### ✅ Step 2: Start Nginx Service
    sudo systemctl start nginx

### ✅ Step 3: Check Service Status
    systemctl status nginx

Expected output:
    Active: active (running)

### ✅ Step 4: Test in Browser
    http://localhost

## 🧠 Key Learnings

- Package managers simplify software management in Linux  
- APT is used in Ubuntu/Debian systems  
- YUM/DNF is used in RedHat/CentOS systems  
- Installing services like nginx is straightforward with package managers  
- Services can be managed using systemctl  
- Real-world cloud environments rely heavily on these tools
