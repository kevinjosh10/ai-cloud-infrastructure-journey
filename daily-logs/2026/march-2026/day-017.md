## 🚀 What I Learned Today

### 🌐 1. Understanding EC2
- EC2 (Elastic Compute Cloud) is a virtual server in the cloud
- It allows full control over a remote Linux machine
- Widely used in real-world cloud and DevOps environments

---

### ⚙️ 2. Launching an EC2 Instance
- Launched a Free Tier Ubuntu 22.04 instance (t2.micro)
- Configured instance settings via AWS Console
- Understood importance of instance type and region selection

---

### 🔑 3. Key Pair Authentication
- Created and downloaded a `.pem` key file
- Applied secure permissions:
  chmod 400 day17-key.pem
- Learned why SSH requires strict permissions

---

### 🔗 4. Connecting via SSH
- Connected using:
  ssh -i day17-key.pem ubuntu@<public-ip>
- Learned SSH authentication and host verification
- Debugged real issues:
  - Wrong directory
  - File not found
  - Permission errors
  - Missing `@` in SSH command
  - Windows vs Linux filesystem differences

---

### 🧠 5. Navigating Linux in the Cloud
- Logged into a remote Ubuntu server
- Ran core commands:
  uname -a     → system/kernel info
  df -h        → disk usage
  free -h      → memory usage

---

### 📊 6. Monitoring with htop
- Installed:
  sudo apt update
  sudo apt install htop
- Used `htop` to monitor:
  - CPU usage
  - RAM usage
  - Active processes

---

### 🔧 7. Real Debugging Experience
- Learned about `/mnt/c` (Windows mount in Linux)
- Understood why chmod fails on Windows filesystem
- Fixed issue by moving `.pem` file to Linux home directory
- Practiced real-world debugging like a cloud engineer

---

### 🛑 8. Instance Management
- Learned how to stop EC2 instance after use
- Understood:
  - Stop vs Terminate
  - Public IP changes on restart
