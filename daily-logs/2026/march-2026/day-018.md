## 🔑 Key Concepts Learned

### 1. SSH Key-Based Authentication
- Generated an SSH key pair using:
  ssh-keygen -t ed25519 -C "your-email@example.com"

- Understood asymmetric encryption:
  - Private Key (id_ed25519) → kept secure, never shared  
  - Public Key (id_ed25519.pub) → shared with servers  

- Authentication flow:
  - Server stores public key  
  - Client proves identity using private key  

---

### 2. SSH File Structure
- ~/.ssh/id_ed25519 → Private key  
- ~/.ssh/id_ed25519.pub → Public key  
- ~/.ssh/authorized_keys → Allowed keys on server  

- EC2 .pem keys follow the same concept  

---

### 3. Connecting to Remote Servers (EC2)
- SSH connection:
  ssh -i mykey.pem ubuntu@<ec2-public-ip>

- Key learnings:
  - Default user depends on OS (ubuntu, ec2-user)
  - SSH uses port 22
  - Fix permission issues:
    chmod 400 mykey.pem

---

### 4. SSH Config File (Productivity Boost)
- File:
  ~/.ssh/config

- Example:
  Host myec2
      HostName <ec2-public-ip>
      User ubuntu
      IdentityFile ~/.ssh/mykey.pem

- Connect using:
  ssh myec2

- Simplifies managing multiple servers  

---

### 5. Secure File Transfer (SCP)
- Local → EC2:
  scp -i mykey.pem file.txt ubuntu@ec2-ip:~/

- EC2 → Local:
  scp -i mykey.pem ubuntu@ec2-ip:~/file.txt .

- SCP = secure file transfer over SSH  

---

### 6. GitHub SSH Integration
- Added public key to GitHub  
- Tested with:
  ssh -T git@github.com  

- Benefits:
  - No repeated login credentials  
  - Secure Git operations  

---

### 7. SSH Security Best Practices
- Never share private keys  
- Use proper permissions (chmod 400)  
- Prefer key-based authentication  
- Never upload keys to GitHub
