## 🎯 Special Variables (Arguments Handling)

### 🔹 Common Variables
- `$1` → First argument  
- `$2` → Second argument  
- `$@` → All arguments  
- `$#` → Number of arguments  
- `$0` → Script name  

---

### 🔹 Example
    #!/bin/bash

    echo "Script: $0"
    echo "First: $1"
    echo "Second: $2"
    echo "Total args: $#"
    echo "All args: $@"

## 🛡️ Safe Script Execution (Production Practice)

### 🔹 Strict Mode
    set -e   # exit on error
    set -u   # undefined variable error
    set -x   # debug mode

---

### 🔹 Best Practice
    #!/bin/bash
    set -euo pipefail

---

### ⚡ Outcome
- Prevented silent script failures  
- Improved debugging visibility  
- Adopted production-level scripting standards

## 🧪 Practice Script — Log Analyzer (Script 2/5)

### 🔹 Script Created
    #!/bin/bash

    set -euo pipefail

    analyze_logs() {
        file=$1

        if [ ! -f "$file" ]; then
            echo "File not found!"
            exit 1
        fi

        errors=$(grep -c "ERROR" "$file" || true)
        warns=$(grep -c "WARN" "$file" || true)
        infos=$(grep -c "INFO" "$file" || true)

        echo "------ Log Summary ------"
        echo "File: $file"
        echo "ERROR count: $errors"
        echo "WARN count: $warns"
        echo "INFO count: $infos"
    }

    if [ $# -eq 0 ]; then
        echo "Usage: $0 <logfile>"
        exit 1
    fi

    analyze_logs "$1"

---

### ⚡ Outcome
- Built a real-world log analysis tool  
- Used functions for modular design  
- Handled errors and user input  
- Simulated DevOps monitoring workflow
