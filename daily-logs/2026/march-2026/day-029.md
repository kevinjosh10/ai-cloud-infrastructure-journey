## 💻 Understanding IPv4 Addressing

Today I learned the fundamentals of **IPv4 addressing**, which is a core concept in networking and essential for Cloud, DevOps, and Infrastructure Engineering.

An IPv4 address is a **32-bit number**, divided into **4 octets (8 bits each)**. Each octet ranges from **0 to 255**, allowing over 4 billion unique addresses.

Example:
192.168.1.100

---

## 🧠 IPv4 Structure

- Total Bits: **32 bits**
- Divided into: **4 octets**
- Each octet: **8 bits**
- Range per octet: **0–255**

👉 8 bits = 2⁸ = 256 values → (0–255)

---

## 🏠 Private IP Address Ranges (RFC 1918)

These IP ranges are reserved for internal use and must be memorized:

- 10.0.0.0 – 10.255.255.255 → Large networks  
- 172.16.0.0 – 172.31.255.255 → Medium networks  
- 192.168.0.0 – 192.168.255.255 → Small/home networks  

👉 These are **NOT routable on the internet**

---

## 🌍 Public vs Private IP Addresses

### 🔒 Private IP
- Used within local networks  
- Not accessible from the internet  
- Example: 192.168.1.5  

### 🌐 Public IP
- Globally unique  
- Assigned by ISPs  
- Accessible over the internet  
- Example: 8.8.8.8  

---

## 🔁 NAT (Network Address Translation)

NAT allows multiple devices with private IP addresses to access the internet using a single public IP.

Example:
192.168.1.10 → Router (NAT) → Public IP → Internet

---

## 🔁 Loopback Address

- Address: **127.0.0.1**  
- Also known as **localhost**  
- Always refers to your own machine  

Used for testing and local development.

---

## 🧪 Practice — Identifying IP Types

- 10.0.1.5 → Private  
- 8.8.8.8 → Public  
- 172.20.0.1 → Private  
- 203.0.113.1 → Public  

---

## ⚡ Quick Identification Rules

- Starts with **10** → Private  
- Starts with **192.168** → Private  
- Starts with **172.16–31** → Private  
- Anything else → Public
