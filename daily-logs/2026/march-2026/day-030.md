## 💻 Understanding Subnetting & CIDR

Today I learned **subnetting**, one of the most critical skills in networking. Subnetting allows us to divide a large network into smaller, manageable sub-networks.

This is essential in **Cloud, DevOps, and Infrastructure Engineering** for designing scalable and secure systems.

CIDR (**Classless Inter-Domain Routing**) replaces old class-based networking and gives flexibility in defining network sizes.

Example:
192.168.1.0/24

---

## 🧠 CIDR Notation

CIDR format:
IP Address / Prefix Length

Example:
192.168.1.0/24

- **/24 → First 24 bits = Network**
- **Remaining 8 bits = Hosts**

👉 Total bits in IPv4 = 32  
👉 Host bits = 32 - CIDR  

---

## 📊 CIDR & Host Calculation

Formula:

- Total IPs = 2^(32 - CIDR)  
- Usable Hosts = Total IPs - 2  

Example:
- /24 → 2⁸ = 256 IPs → 254 usable  

---

## 📌 Common CIDR Blocks (Must Memorize)

- /32 → 1 IP  
- /31 → 2 IPs (special case)  
- /30 → 4 IPs (2 usable)  
- /29 → 8 IPs (6 usable)  
- /28 → 16 IPs (14 usable)  
- /27 → 32 IPs (30 usable)  
- /26 → 64 IPs (62 usable)  
- /25 → 128 IPs (126 usable)  
- /24 → 256 IPs (254 usable)  
- /16 → 65536 IPs  

---

## 🔍 Subnetting Example

Given:
192.168.1.0/24

- Network Address: 192.168.1.0  
- First Host: 192.168.1.1  
- Last Host: 192.168.1.254  
- Broadcast Address: 192.168.1.255  

---

## ⚡ Block Size Method (Fast Subnetting)

Formula:
Block Size = 256 - Subnet Mask (last relevant octet)

Example:
/26 → 255.255.255.192  

Block Size:
256 - 192 = 64  

Subnets:
- 192.168.1.0  
- 192.168.1.64  
- 192.168.1.128  
- 192.168.1.192  

---

## ⚡ 10-Second Subnetting Method

1. Identify CIDR → recall block size  
2. Find changing octet  
3. Locate nearest lower multiple  
4. Add block size → get range  

Example:
192.168.1.70/26  

- Block size = 64  
- Range = 64 – 127  

Answer:
- Network: 192.168.1.64  
- Broadcast: 192.168.1.127  
- First: 192.168.1.65  
- Last: 192.168.1.126  

---

## 🧠 Memory Hack (Important)

Memorize this:

- /25 → 128  
- /26 → 64  
- /27 → 32  
- /28 → 16  
- /29 → 8  
- /30 → 4  

👉 This eliminates the need for binary calculations.
