## 🎯 Objective

Build a real-world **EC2 Lifecycle Management CLI Tool** using Python.

Learn how to:
- Automate EC2 operations (start, stop, reboot, terminate)
- Target resources using **Instance IDs and Tags**
- Implement **safe execution practices** (confirmation prompts, dry-run)
- Handle AWS errors using `ClientError`
- Introduce **YAML-based scheduling** for automation

---

## ✅ What I Completed

### 1️⃣ Built EC2 Lifecycle CLI Tool
- Created a Python script using boto3
- Implemented core EC2 actions:
  - Start instances  
  - Stop instances  
  - Reboot instances  
  - Terminate instances  
- Used `argparse` to create a CLI interface

Example:
    python ec2_lifecycle.py stop --instance-ids i-12345

---

### 2️⃣ Implemented Tag-Based Operations
- Learned how to filter EC2 instances using tags
- Enabled commands like:

    python ec2_lifecycle.py stop --tag Environment=dev --region us-east-1

- Used `describe_instances` with filters
- Extracted instance IDs dynamically

---

### 3️⃣ Added Safety Features (Production-Level Thinking)

#### Confirmation Prompt
- Added manual confirmation before destructive actions
- Prevents accidental termination

Example:
    ⚠️ Are you sure you want to TERMINATE instances? (yes/no)

#### Dry-Run Mode
- Implemented `--dry-run` flag
- Allows testing without actual execution

Example:
    python ec2_lifecycle.py terminate --tag Environment=dev --dry-run

---

### 4️⃣ Implemented Error Handling
- Used `ClientError` from boto3
- Handled AWS DryRun responses properly

Learned:
- `DryRunOperation` → permissions are correct  
- Other errors → actual failures  

Improved reliability of automation script

---

### 5️⃣ Added YAML-Based Scheduling
- Introduced configuration-driven automation
- Created a `schedule.yaml` file

Example:
    schedule:
      - action: stop
        tag: Environment=dev
        time: "22:00"

      - action: start
        tag: Environment=dev
        time: "08:00"

- Parsed YAML using `pyyaml`
- Compared current time with schedule

---

### 6️⃣ Understood Automation Workflow
- Combined CLI + YAML + AWS API calls
- Simulated real-world automation system
- Learned how scheduling works conceptually

Real-world usage:
- Cron jobs (Linux)
- AWS Lambda + EventBridge
