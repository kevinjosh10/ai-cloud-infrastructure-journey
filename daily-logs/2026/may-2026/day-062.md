## 🎯 Objective

Build a production-style S3 backup and sync tool using Python and AWS.  
Learn how to automate backups, optimize storage costs, and implement secure cloud storage practices.

---

## ✅ What I Completed

### 1️⃣ Built S3 Backup Upload Tool
- Created a Python script using boto3
- Uploaded local directory to S3 bucket
- Implemented timestamp-based versioning
- Structured S3 object keys for organized backups

Example:
    2026-05-02_18-30-00/file.txt  

---

### 2️⃣ Implemented Incremental Backup Logic
- Avoided re-uploading unchanged files
- Generated MD5 checksum for local files
- Compared with S3 ETag values

Concept:
- Upload only if file has changed
- Skip duplicate files to save time and cost

---

### 3️⃣ Learned S3 ETag & MD5 Comparison
- Understood ETag represents file hash (for small files)
- Used hashlib to generate MD5 checksum
- Compared local vs cloud file integrity

---

### 4️⃣ Added Server-Side Encryption
- Applied AES256 encryption during upload

Used:
    ServerSideEncryption = 'AES256'

- Ensured all data stored securely in S3
- Learned basic encryption practices in cloud

---

### 5️⃣ Automated S3 Lifecycle Policy
- Programmatically configured lifecycle rules

Rules:
- Move to Standard-IA after 30 days  
- Move to Glacier after 90 days  
- Delete after 365 days  

- Learned cost optimization strategies in cloud storage

---

### 6️⃣ Built Backup Restore (Download Feature)
- Listed objects using prefix (timestamp)
- Downloaded specific backup versions
- Recreated folder structure locally

---

### 7️⃣ Added CLI Support
- Used argparse for command-line interface

Commands:
    python s3_tool.py upload --dir data  
    python s3_tool.py download --backup <timestamp>  

- Made tool usable like real DevOps utility

---

### 8️⃣ Prepared GitHub Project

  - requirements.txt
