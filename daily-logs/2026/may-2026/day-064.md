# Day 064 – Git Internals: Understanding the Fundamentals

## 🎯 Objective

In 2 hours, build a strong foundation of how Git works internally.

Learn:
- Git’s **data model (SHA-1)**
- The 3 areas: **Working Directory, Staging Area, Repository**
- Use of `git add -p`
- Understanding `git diff`
- Visualizing history using `git log`
- Deep clarity on **HEAD and Detached HEAD**

---

## ⏱️ 2-Hour Learning Plan

### 🕐 Hour 1 – Core Concepts + Staging

#### 1️⃣ Understand Git Data Model (20 mins)
- Git stores everything as **SHA-1 hashes**
- Each commit = **snapshot of project**
- Even a small change → new hash

👉 Key idea:
Git is a **content-addressable database**

---

#### 2️⃣ Learn the 3 Areas (20 mins)

- **Working Directory** → where you edit files  
- **Staging Area (Index)** → where you prepare commits  
- **Repository (.git)** → where history is stored  

Flow:
    Working Directory → Staging Area → Repository

---

#### 3️⃣ Practice `git add -p` (20 mins)

Instead of:
    git add .

Use:
    git add -p

Learn:
- `y` → stage chunk  
- `n` → skip  
- `s` → split changes  
- `e` → manual edit  

👉 Goal:
Understand **how to stage only specific changes**

---

### 🕑 Hour 2 – Diff, Logs & HEAD

#### 4️⃣ Learn `git diff` (20 mins)

Commands:
    git diff
    git diff --staged

Understand:
- Unstaged vs staged changes  
- What exactly will go into commit  

---

#### 5️⃣ Visualize History (20 mins)

Command:
    git log --oneline --graph --all

Set alias:
    git config --global alias.lg "log --oneline --graph --all"

Use:
    git lg

👉 Goal:
Understand **branch structure and commit flow**

---

#### 6️⃣ Master HEAD (20 mins)

- HEAD = **pointer to current commit**

Normal:
    HEAD → main → latest commit

Detached:
    git checkout <commit_hash>

Now:
    HEAD → specific commit (no branch)

⚠️ Risk:
- New commits can be lost

Fix:
    git switch -c new-branch

👉 Goal:
Understand **where you are in Git at all times**

### Step 2: Create 5 Commits

    echo "v2" >> app.txt
    git commit -am "v2"

    echo "v3" >> app.txt
    git commit -am "v3"

    echo "v4" >> app.txt
    git commit -am "v4"

    echo "v5" >> app.txt
    git commit -am "v5"

---

### Step 3: Inspect History
    git lg

---

### Step 4: Inspect a Commit
    git show <commit_hash>

---

### Step 5: Try Detached HEAD
    git checkout <old_commit_hash>
    git status

Return:
    git switch main

---

## 🧠 Key Concepts Learned

- Git stores data using **SHA-1 hashes**  
- Commits are **snapshots, not diffs**  
- Staging area gives control over commits  
- `git add -p` enables clean commits  
- `git diff` shows exact changes  
- `git log --graph` shows commit structure  
- HEAD tracks **current position in Git**  
- Detached HEAD can lead to **lost commits**
