## 🎯 Objective

Understand how real-world teams manage code using Git branching strategies.

Learn how to:
- Perform core branch operations (`branch`, `switch`, `-c`)
- Work with different branching models (Git Flow, GitHub Flow, Trunk-Based Development)
- Understand when and why each strategy is used
- Simulate real-world collaboration workflows using branches

---

## ✅ What I Completed

### 1️⃣ Practiced Core Git Branch Operations
- Created new branches using:

    git branch feature-login

- Switched between branches:

    git switch feature-login

- Created and switched in one step:

    git switch -c feature-payment

- Listed all branches:

    git branch

- Deleted branches after merging:

    git branch -d feature-login

---

### 2️⃣ Understood Git Flow (Traditional Model)
- Learned structured branching system:

  - main → Production  
  - develop → Integration  
  - feature/* → New features  
  - release/* → Pre-release testing  
  - hotfix/* → Emergency fixes  

- Practiced workflow:
  - Created feature branch from develop  
  - Merged feature into develop  
  - Created release branch  
  - Understood hotfix flow from main  

- Observed that Git Flow is:
  - Powerful but complex  
  - Used in large teams and enterprise systems  

---

### 3️⃣ Learned GitHub Flow (Modern Workflow)
- Understood simpler branching strategy:

  - main → Always production-ready  
  - feature branches → Short-lived  

- Practiced workflow:
  - Created branch from main  
  - Made changes and committed  
  - Pushed to remote repository  
  - Opened Pull Request (PR)  
  - Merged after review  

- Key takeaway:
  - main branch must always be deployable  

---

### 4️⃣ Explored Trunk-Based Development
- Learned advanced workflow used by top tech companies
- Key idea:
  - main (trunk) is the central branch  
  - branches are very short-lived  

- Practiced:
  - Creating small branches  
  - Making small commits  
  - Merging frequently  

- Learned principles:
  - Frequent integration  
  - Small changes  
  - Use of feature flags  
  - Continuous deployment  

---

### 5️⃣ Compared Branching Strategies

| Strategy            | Complexity | Speed | Usage |
|--------------------|----------|------|------|
| Git Flow           | High     | Slow | Enterprises |
| GitHub Flow        | Medium   | Fast | Most modern teams |
| Trunk-Based Dev    | Low (concept) / High (discipline) | Very Fast | Top tech companies |

### 7️⃣ Simulated Real Workflow Locally
- Initialized a repository:

    git init

- Created initial commit:

    echo "v1" > app.txt
    git add .
    git commit -m "initial commit"

- Created feature branch:

    git switch -c feature-update
    echo "new feature" >> app.txt
    git commit -am "added feature"

- Merged back to main:

    git switch main
    git merge feature-update
