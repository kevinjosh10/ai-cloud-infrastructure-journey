## ✅ What I Completed

### 1️⃣ Learned Git Merge

#### Fast-Forward Merge
- Occurs when there is no divergence between branches  
- Simply moves the branch pointer forward  
- Maintains a linear history  

#### Recursive Merge (Merge Commit)
- Happens when branches have diverged  
- Creates a new merge commit  
- Preserves full branch history  

Understood:
- Merge is safe and commonly used in team environments  
- Helps visualize branch structure clearly  

---

### 2️⃣ Learned Git Rebase

- Reapplies commits from one branch onto another  
- Creates a clean, linear history  
- Rewrites commit history (new commit IDs)  

Command used:
    git rebase main  

Understood:
- Rebase is useful for cleaning history before merging  
- Should not be used on shared/public branches  

---

### 3️⃣ Understood Merge vs Rebase

- Merge preserves complete history with merge commits  
- Rebase creates a cleaner, linear history  

When to use:
- Merge → team collaboration, shared branches  
- Rebase → personal branches, before pull requests  

---

### 4️⃣ Learned Interactive Rebase

Command used:
    git rebase -i HEAD~3  

Practiced:
- Squashing multiple commits into one  
- Rewording commit messages  
- Reordering commits  
- Dropping unnecessary commits  

Understood:
- Helps maintain a professional and readable commit history  

---

### 5️⃣ Learned Merge Conflicts

Understood when conflicts occur:
- Same file modified in multiple branches  
- Same lines edited differently  

Observed conflict markers:
    <<<<<<< HEAD
    =======
    >>>>>>> feature  

Steps followed:
- Manually resolved conflicts in file  
- Staged changes using `git add`  
- Continued process using:
    git merge --continue  
    git rebase --continue  

Also learned:
- How to abort operations using:
    git merge --abort  
    git rebase --abort
