## ✅ What I Completed

### 1️⃣ Learned Conventional Commits Format

Learned the standard commit message structure:

    type(scope): description

Understood:
- `type` → kind of change
- `scope` → affected module/service
- `description` → short explanation

Example:
    feat(ec2): add multi-region instance listing

---

### 2️⃣ Learned Common Conventional Commit Types

#### feat
- Used for new features

Example:
    feat(terraform): add VPC module

---

#### fix
- Used for bug fixes

Example:
    fix(s3): handle bucket not found error

---

#### docs
- Documentation updates

Example:
    docs: update README with usage examples

---

#### style
- Formatting-only changes
- No logic changes

Example:
    style(python): reformat code with black

---

#### refactor
- Improves code structure without changing functionality

Example:
    refactor(auth): simplify token validation

---

#### test
- Adds or updates tests

Example:
    test(api): add login endpoint tests

---

#### chore
- Maintenance tasks

Example:
    chore: update dependencies

---

#### ci
- CI/CD workflow changes

Example:
    ci(github-actions): add terraform validation workflow

---

### 9️⃣ Learned .gitignore Pattern Rules

#### Wildcard Matching
    *

Matches multiple files.

Example:
    *.log

Ignores all `.log` files.

---

#### Negation
    !

Used to exclude a file from ignore rules.

Example:
    !important.log

---

#### Any Directory Depth
    **/

Matches folders recursively.

Example:
    **/__pycache__/

---

#### Directory Matching

Example:
    venv/

Ignores entire directory.

---

#### File Matching

Example:
    *.tfstate

Ignores all Terraform state files.
