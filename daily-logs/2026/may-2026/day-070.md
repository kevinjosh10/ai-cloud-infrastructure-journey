### 3️⃣ Mastered Git Rebase (Very Important)

Learned how rebasing creates cleaner and more professional commit history.

Command used:
    git rebase main

Before Rebase:
    A --- B --- C
                 \
                  D --- E

After Rebase:
    A --- B --- C --- D' --- E'

Learned:
- Rebase rewrites commit history
- Commits are replayed on top of another branch
- Produces linear and cleaner commit structure
- Preferred in many professional engineering teams

Understood:
- Rebase improves readability
- Easier debugging and rollback
- Cleaner Git logs for large projects

Important rule learned:
- Never rebase shared public branches

---

### 4️⃣ Practiced Interactive Rebase

Learned how to clean and rewrite commit history professionally.

Command used:
    git rebase -i HEAD~3

Editor opened with:
    pick a1b2c3 fixed stuff
    pick d4e5f6 test
    pick g7h8i9 final

Learned interactive rebase actions:
- pick → keep commit
- reword → rename commit
- squash → combine commits
- fixup → combine without keeping message
- drop → remove commit

Modified example:
    reword a1b2c3 fixed stuff
    squash d4e5f6 test
    squash g7h8i9 final

Final result:
    feat: add authentication system

Understood:
- Interactive rebase cleans messy histories
- Professional repositories maintain readable commits
- Clean commit history improves long-term maintenance

### 6️⃣ Learned Safe Force Push Workflow

Understood why force pushing becomes necessary after rebasing.

Command used:
    git push --force-with-lease

Learned:
- Rebasing changes commit hashes
- GitHub rejects normal pushes after rewritten history
- Force push updates remote history

Compared:
    git push --force
vs
    git push --force-with-lease

Understood:
- `--force-with-lease` is safer
- Prevents accidentally overwriting teammate commits
- Recommended in collaborative environments
