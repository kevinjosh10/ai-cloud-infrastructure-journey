### 1️⃣ Learned EC2 Instance Naming Convention

Understood how AWS names EC2 instances.

Example:
    m5.xlarge

Breakdown:
- `m` → Instance family  
- `5` → Generation/version  
- `xlarge` → Instance size  

Learned:
- Family determines workload optimization
- Newer generations provide better performance
- Larger sizes provide more CPU, RAM, and bandwidth

---

### 2️⃣ Memorized Important EC2 Instance Families

#### T Family — Burstable Performance
Examples:
    t3.micro
    t3.small

Use Cases:
- Learning AWS
- Development environments
- Small applications
- Lightweight servers

Key Points:
- Low-cost instances
- Uses CPU credit system
- Good for temporary CPU bursts

---

#### M Family — General Purpose
Examples:
    m5.large
    m6i.large

Use Cases:
- Backend applications
- Web servers
- APIs
- Docker workloads

Key Points:
- Balanced CPU and memory
- Default safe choice for most workloads
- Common in production environments

---

#### C Family — Compute Optimized
Examples:
    c5.large
    c5.2xlarge

Use Cases:
- CPU-intensive applications
- Data processing
- CI/CD pipelines
- Batch jobs

Key Points:
- Higher compute performance
- Good for multi-threaded workloads
- Optimized for processing power

---

#### R Family — Memory Optimized
Examples:
    r5.large
    r6i.large

Use Cases:
- Databases
- Redis caching
- Big data analytics
- In-memory applications

Key Points:
- Large RAM capacity
- Suitable for memory-heavy systems

---

#### P Family — GPU Training
Example:
    p3.2xlarge

Use Cases:
- Deep learning training
- AI model development
- TensorFlow/PyTorch workloads

Key Points:
- Uses NVIDIA GPUs
- Very powerful but expensive
- Designed for ML training

---

#### G Family — GPU Inference
Example:
    g4dn.xlarge

Use Cases:
- AI inference
- Computer vision
- Video processing
- Model serving

Key Points:
- More affordable GPU option
- Useful for deployed AI applications

---

#### INF Family — Inferentia Optimized
Example:
    inf1.xlarge

Use Cases:
- ML inference at scale
- Production AI deployments

Key Points:
- AWS custom AI hardware
- Lower inference cost
- Optimized for serving trained models

---

#### I Family — Storage Optimized
Example:
    i3.large

Use Cases:
- High-speed databases
- Storage-intensive workloads
- Analytics systems

Key Points:
- Fast local SSD storage
- High IOPS performance

### 4️⃣ Understood Core EC2 Instance Attributes

#### vCPU
- Represents virtual CPUs
- Determines processing capability

Learned:
- More vCPUs = better parallel processing
- Important for compute-heavy applications

---

#### Memory (RAM)
- Determines how much data applications can hold

Learned:
- Databases and AI models require large memory
- Insufficient RAM can crash workloads

---

#### Network Bandwidth
- Determines communication speed between systems

Important For:
- Distributed systems
- Cloud applications
- Data transfer workloads

---

#### EBS Optimization
Learned:
- Dedicated bandwidth for EBS storage traffic
- Improves storage performance
- Important for databases and production systems

### 6️⃣ Learned Instance Store vs EBS-backed Storage

#### Instance Store
Key Points:
- Temporary storage
- Data lost after stop/terminate
- Very high-speed local storage

Used For:
- Cache
- Temporary processing data

---

#### EBS-backed Storage
Key Points:
- Persistent storage
- Data survives stop/reboot
- Most common EC2 storage type

Used For:
- Servers
- Databases
- Production systems
