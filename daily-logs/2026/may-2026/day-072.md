### 2️⃣ Understood AMI Structure

Studied the three major AMI components.

---

#### Root Volume Snapshot

Learned:
- Stores the operating system
- Stores installed software
- Preserves filesystem state

Examples:
- Ubuntu OS
- Nginx installation
- Python packages
- Configuration files

Understood:
- AWS stores AMIs using EBS snapshots
- Snapshots preserve complete server state

---

#### Launch Permissions

Learned:
- Permissions define who can use an AMI

Types:
- Private
- Shared
- Public

Understood:
- Organizations securely share internal AMIs
- Public AMIs are available globally
- Access control improves infrastructure security

---

#### Block Device Mapping

Learned:
- Defines storage configuration
- Controls attached EBS volumes

Examples:
- Root volume size
- Additional storage volumes
- Device mappings

Understood:
- Infrastructure launches consistently every time
- Storage settings become reusable

### 9️⃣ Compared AMIs vs Snapshots

Clarified common beginner confusion.

Comparison learned:

| Feature | AMI | Snapshot |
|---|---|---|
| Bootable | Yes | No |
| Launch EC2 | Yes | No |
| Stores OS | Yes | Yes |
| Contains metadata | Yes | No |
| Used for cloning | Yes | Partial |

Understood:
- Snapshots are storage backups
- AMIs are complete launch templates
- AMIs include configuration and boot settings
