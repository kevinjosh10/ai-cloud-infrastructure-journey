### 1️⃣ Learned What Amazon EBS Is

Understood that Amazon Elastic Block Store (EBS) is persistent block storage designed for EC2 instances.

Learned:

- Volumes can store OS files, applications, and databases
- Data persists even if EC2 stops or reboots

### 2️⃣ Learned EBS Volume Types

Studied the different EBS storage types available in AWS.

Volume types explored:

| Volume Type | Storage Type | Best Use Case |
|---|---|---|
| gp3 | SSD | General workloads |
| io1/io2 | SSD | Databases & critical apps |
| st1 | HDD | Big data & throughput |
| sc1 | HDD | Cold/archive storage |

Learned:

- SSD volumes provide fast random access
- HDD volumes provide cheaper large-capacity storage

Understood:

Use cases studied:

### 8️⃣ Formatted New EBS Volumes

Learned that newly created EBS volumes are empty and require formatting.

Command practiced:

- Formatting prepares the disk for file storage

Understood:

- Filesystems organize data on disks

### 🔟 Learned Persistent Mounting Using /etc/fstab

Studied how to automatically remount EBS volumes after reboot.

Command practiced:
    echo '/dev/xvdf /data ext4 defaults,nofail 0 0' | sudo tee -a /etc/fstab

Learned:
- `/etc/fstab` controls automatic filesystem mounting
- Without fstab configuration, mounts disappear after reboot

Fields understood:

| Field | Meaning |
|---|---|
| /dev/xvdf | Device |
| /data | Mount point |
| ext4 | Filesystem |
| defaults | Default options |
| nofail | Prevent boot failure |
| 0 0 | Backup/check values |

Understood:
- Persistent mounting is essential in production systems
- Incorrect fstab configuration can break Linux boot processes
