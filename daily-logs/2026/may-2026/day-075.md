### 2️⃣ Learned On-Demand EC2 Pricing

Studied the default EC2 pricing model in AWS.

Learned:
- Pay only for usage
- No long-term commitment
- Billed per second/hour
- Highest pricing compared to other models

Characteristics explored:
- Launch anytime
- Stop anytime
- Maximum flexibility

Best use cases studied:
- Development environments
- Testing systems
- Temporary workloads
- Unpredictable traffic
- New applications

Visualized:
    Start EC2
        ↓
   Use Resources
        ↓
     Pay Only
    For Usage

Understood:
- On-Demand is best for unpredictable workloads
- Flexibility comes with higher cost
- Ideal for learning and experimentation environments

---

### 3️⃣ Learned Reserved Instances (RIs)

Studied AWS Reserved Instances and long-term infrastructure commitments.

Learned:
- Reserved Instances require:
  - 1-year commitment
  OR
  - 3-year commitment
- Discounts can reach up to 72%

Understood:
- AWS rewards predictable infrastructure usage
- Long-term commitments reduce operational uncertainty for AWS

Visualized:
    Long-Term Commitment
             ↓
       AWS Discount
             ↓
      Lower EC2 Cost

Best use cases explored:
- Stable production systems
- Enterprise applications
- Long-running backend services
- Databases with predictable workloads

Understood:
- Reserved pricing is effective for steady workloads
- Production environments often use RIs for cost reduction

---

### 4️⃣ Learned Standard Reserved Instances

Studied Standard Reserved Instances in detail.

Learned:
- Standard RIs provide maximum discounts
- Limited flexibility after purchase
- Instance family changes are difficult

Example studied:
    Reserved for:
        m5.large

Cannot easily switch to:
        t3.large

Best use cases explored:
- Stable enterprise infrastructure
- Predictable production environments
- Applications with fixed requirements

Understood:
- Standard RIs prioritize savings over flexibility
- Stable architectures benefit most from this pricing model

---

### 5️⃣ Learned Convertible Reserved Instances

Studied flexible Reserved Instance options.

Learned:
- Convertible RIs allow:
  - Instance family changes
  - OS changes
  - Instance size modifications

Tradeoff explored:
- Slightly lower discount than Standard RIs
- Much greater flexibility

Best use cases:
- Evolving infrastructure
- Growing startups
- Architecture modernization projects

Visualized:
    Convertible RI
          ↓
   Flexible Changes
          ↓
 Better Adaptability

Understood:
- Cloud infrastructure changes frequently
- Flexibility is valuable in dynamic environments

---

### 6️⃣ Learned Scheduled Reserved Instances

Studied time-based EC2 reservation models.

Learned:
- Capacity reserved only during specific schedules
- Example:
  - Weekdays
  - 9 AM → 5 PM

Best use cases explored:
- Office applications
- Educational labs
- Predictable business-hour systems

Understood:
- Not all workloads require 24/7 infrastructure
- Scheduled reservations improve cost efficiency

---

### 7️⃣ Learned AWS Savings Plans

Studied modern AWS pricing optimization models.

Learned:
- Savings Plans are more flexible than Reserved Instances
- Commit to spending amount instead of specific instances

Example explored:
    Commit:
      $10/hour usage

AWS automatically applies discounts.

Understood:
- Savings Plans simplify cloud cost optimization
- AWS designed Savings Plans to replace RI complexity

---

### 8️⃣ Learned Compute Savings Plans

Focused on the most flexible Savings Plan model.

Learned:
- Works across:
  - EC2
  - Lambda
  - Fargate

Flexibility explored:
- Change instance families
- Change regions
- Change operating systems

Best use cases:
- Modern cloud-native environments
- Dynamic production infrastructure
- Multi-service AWS architectures

Visualized:
    Compute Savings Plan
          ↓
  Flexible Infrastructure
          ↓
    Automatic Discounts

Understood:
- Compute Savings Plans are highly flexible
- Many organizations prefer them over Reserved Instances

---

### 9️⃣ Learned EC2 Savings Plans

Studied EC2-specific Savings Plans.

Learned:
- Locked to EC2 instance family within a region
- Slightly cheaper than Compute Savings Plans

Best use cases:
- Stable EC2 workloads
- Predictable compute usage

Understood:
- Less flexibility provides better pricing
- Tradeoff exists between adaptability and cost savings

---

### 🔟 Learned Spot Instances

Studied AWS Spot pricing and unused compute capacity.

Learned:
- Spot Instances use unused AWS infrastructure
- Discounts can reach up to 90%
- AWS can reclaim instances anytime

Critical concept learned:
- AWS provides only 2-minute interruption warning

Visualized:
    Unused AWS Capacity
             ↓
       Spot Instance
             ↓
       Huge Discount

Understood:
- Spot Instances are extremely cost effective
- Reliability tradeoffs must be considered carefully
