### 🔟 Learned IMDSv2 Security Improvements

Studied AWS secure metadata access model.

Learned:
- IMDSv2 uses session tokens
- Token-based authentication improves security
- Prevents many metadata exploitation techniques

Workflow studied:

Step 1 — Generate Token

```bash
TOKEN=$(curl -X PUT "http://169.254.169.254/latest/api/token" \
-H "X-aws-ec2-metadata-token-ttl-seconds: 21600")
```

Step 2 — Use Token

```bash
curl -H "X-aws-ec2-metadata-token: $TOKEN" \
http://169.254.169.254/latest/meta-data/instance-id
```

Understood:
- Modern production systems prefer IMDSv2 only
- Security-conscious environments disable IMDSv1 entirely

Visualized:
    Request Token
          ↓
  Authenticated Access
          ↓
   Secure Metadata
